# skills-manage 0.9.3

## English

Feature release: the central skills directory is now configurable from the UI, and downstream commands honour the configured path.

### Highlights

- New "Central Skills Directory" card in **Settings**: edit, save, and trigger a rescan in one step.
- `set_central_skills_dir` Tauri command persists the path on the `central` agent record (with `~` expansion) and is wired to the new UI.
- Marketplace install (`install_marketplace_skill`, new `write_skill_to_central`) and project discovery (`start_project_scan`) now read the configured central directory instead of the hardcoded default.
- Central library empty-state hint and Marketplace preview-install path now reflect the configured directory.
- DB seed `seed_builtin_agents` no longer overwrites a user-customised `global_skills_dir` for the `central` agent on every startup.

### Packages

- `skills-manage_0.9.3_macos_universal.dmg`
- `skills-manage_0.9.3_macos_universal.zip`
- `skills-manage_0.9.3_macos_universal.tar.gz`
- `skills-manage_0.9.3_windows_x64.msi`
- `skills-manage_0.9.3_windows_x64.zip`

### Notes

- Builds on the recursive central-directory scanning shipped in `0.9.2`, so a user-set central directory works at any depth and is automatically respected by Marketplace + Discover flows.
- Desktop builds remain unsigned, and macOS notarization is still deferred.

### Changelog

- Backend
  - `set_central_skills_dir` (`src-tauri/src/commands/settings.rs`) + DB helper `update_central_skills_dir` (`src-tauri/src/db.rs`).
  - `seed_builtin_agents` preserves an existing `central` agent's `global_skills_dir` on `ON CONFLICT` upsert.
  - `install_marketplace_skill` and new `write_skill_to_central` (`src-tauri/src/commands/marketplace.rs`) resolve the central path from DB.
  - `start_project_scan` (`src-tauri/src/commands/discover.rs`) resolves the central path from DB.
- Frontend
  - "Central Skills Directory" card in `src/pages/SettingsView.tsx` with input + Save + post-save rescan.
  - `src/pages/CentralSkillsView.tsx` empty-state hint reads the configured central path.
  - `src/pages/MarketplaceView.tsx` preview install switched from a hardcoded `~/.agents/skills` write to the new `write_skill_to_central` command.
  - i18n: 5 new keys (`settings.centralDir*`).
- Tests
  - Updated `SettingsView.test.tsx` "saves the github pat from settings" to scope its `保存` lookup to the GitHub PAT card so it does not collide with the new central-directory Save button.

## 中文

新功能版本：中央技能目录现在可以在界面里直接配置，所有依赖该路径的命令都会读取最新配置。

### 亮点

- **设置**页新增 "中央技能目录" 卡片：编辑路径 → 保存 → 自动触发重新扫描，一步搞定。
- 新增 `set_central_skills_dir` 后端命令，写入 `central` agent 的路径（支持 `~` 展开），并接入到新 UI。
- 技能市场安装（`install_marketplace_skill`，新增 `write_skill_to_central`）与项目发现（`start_project_scan`）改为读取数据库里配置的中央路径，而不是写死的默认值。
- 中央库的空状态提示和市场预览安装路径都会反映当前配置。
- 启动时的 `seed_builtin_agents` 不再用默认值覆盖用户自定义的 `central` 路径。

### 安装包

- `skills-manage_0.9.3_macos_universal.dmg`
- `skills-manage_0.9.3_macos_universal.zip`
- `skills-manage_0.9.3_macos_universal.tar.gz`
- `skills-manage_0.9.3_windows_x64.msi`
- `skills-manage_0.9.3_windows_x64.zip`

### 说明

- 与 `0.9.2` 的递归扫描搭配：自定义的中央目录可以放在任意层级，技能市场与项目发现也会同步识别。
- 当前桌面构建仍为未签名状态，macOS notarization 继续留待后续版本处理。

### 更新内容

- 后端
  - 新增 `set_central_skills_dir`（`src-tauri/src/commands/settings.rs`）+ DB 辅助函数 `update_central_skills_dir`（`src-tauri/src/db.rs`）。
  - `seed_builtin_agents` 在 `ON CONFLICT` 升级时保留已有 `central` 的 `global_skills_dir`。
  - `install_marketplace_skill` 与新增的 `write_skill_to_central`（`src-tauri/src/commands/marketplace.rs`）改为从 DB 解析中央路径。
  - `start_project_scan`（`src-tauri/src/commands/discover.rs`）从 DB 解析中央路径。
- 前端
  - `src/pages/SettingsView.tsx` 新增 "中央技能目录" 卡片，包含输入框 + 保存按钮 + 保存后自动 rescan。
  - `src/pages/CentralSkillsView.tsx` 空状态提示读取配置的中央路径。
  - `src/pages/MarketplaceView.tsx` 预览安装从写死的 `~/.agents/skills` 改为调用 `write_skill_to_central` 命令。
  - i18n 新增 5 个 key（`settings.centralDir*`）。
- 测试
  - `SettingsView.test.tsx` 中 "saves the github pat from settings" 用例改为先按输入框定位 PAT 卡片再在卡片内查找 `保存` 按钮，避免与新的中央目录保存按钮冲突。
