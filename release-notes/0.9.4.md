# skills-manage 0.9.4

## English

Feature release for the project-discovery scan-roots dialog: users can now add their own scan paths, copy any path to the clipboard, and long paths render with a middle ellipsis instead of overflowing the dialog.

### Highlights

- New "+ Add path" button in the **Discover → Rescan** dialog. Opens a native directory picker; selected paths are persisted alongside the built-in defaults.
- Custom (user-added) rows have a ✕ button to delete them. Built-in defaults remain enable/disable-only.
- Every row now has a copy icon that puts the full path on the clipboard, plus a native `title` tooltip on hover.
- Long paths render with a middle ellipsis (RTL truncation on the head segment + always-visible basename); the dialog has a wider `2xl` max width and proper `min-w-0` constraints so it no longer balloons past its `max-w` when paths are long.
- Persisted scan-roots config upgraded to `{ overrides, custom }`. Legacy `HashMap<path, enabled>` configs are auto-migrated transparently on read.

- `skills-manage_0.9.4_windows_x64.msi`
- `skills-manage_0.9.4_windows_x64.zip`

### Notes

- Scanning still recurses up to `MAX_SCAN_DEPTH = 8` levels under each root, with the existing `node_modules / .git / target / dist / .venv / ...` skip list.
- Desktop builds remain unsigned, and macOS notarization is still deferred.

### Changelog

- Backend
  - New `add_scan_root` and `remove_scan_root` Tauri commands in `src-tauri/src/commands/discover.rs`, with absolute-path / existence / directory validation and dedup against built-in defaults.
  - `ScanRoot` gains an `is_custom: bool` field; new `ScanRootsConfig` struct + `load_scan_roots_config` / `save_scan_roots_config` helpers replace the old bare-`HashMap` persistence.
  - `get_scan_roots` and `set_scan_root_enabled` rewritten on top of the new helpers.
- Frontend
  - `src/components/discover/DiscoverConfigDialog.tsx`: "+ Add path" button (uses `@tauri-apps/plugin-dialog`), per-row copy icon (`navigator.clipboard.writeText`) and ✕ remove icon for custom rows.
  - Path display switched to a head/tail two-span layout with `direction: rtl` truncation and an always-visible basename; native `title` tooltip on hover.
  - Dialog max width bumped from `sm:max-w-lg` to `sm:max-w-xl md:max-w-2xl`; `min-w-0` added to the grid item, section wrapper, and list container so long paths cannot expand the dialog past its cap.
  - `src/stores/discoverStore.ts`: new `addScanRoot` / `removeScanRoot` actions.
  - i18n: 11 new keys (`discover.addPath` / `removePath` / `copyPath` / `copyPathTooltip` / `pathCopied` / `pathCopyError` / `pickDirectoryTitle` / `addPathSuccess` / `addPathError` / `removePathError` / `addPathUnavailable`) in both `en.json` and `zh.json`.
- Tests
  - 3 new Rust tests (`test_add_and_remove_custom_scan_root`, `test_add_scan_root_rejects_invalid_paths`, `test_legacy_config_format_is_parsed`).
  - Existing `ScanRoot` literals in `discover.rs` tests updated for the new `is_custom` field.
  - `src/test/DiscoverView.test.tsx` mock state extended with `addScanRoot` / `removeScanRoot`.

## 中文

新功能版本：项目技能库「重新扫描」对话框支持自定义扫描路径、复制完整路径，长路径以中间省略号显示而不再撑破对话框。

### 亮点

- **重新扫描对话框**新增 「+ 添加路径」 按钮，点击后弹出系统目录选择器；新增路径与默认目录一同持久化展示。
- 自定义路径行右侧有 ✕ 按钮可移除；内置默认目录依然只能启用/禁用。
- 每行扫描根都新增复制图标按钮，点击即把完整路径写入剪贴板并 toast 提示；悬停时还会通过原生 `title` 展示完整路径。
- 长路径以**中间省略号**展示（开头 RTL 截断 + 末级 basename 始终可见）。对话框最大宽度从 `lg` 扩到 `2xl`，并补齐 `min-w-0` 约束，长路径不再把对话框撑出 `max-w` 范围。
- 扫描根持久化格式从 `HashMap<path, enabled>` 升级为 `{ overrides, custom }`，旧配置自动透明迁移。
