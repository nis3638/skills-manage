# skills-manage 0.9.2

## English

Maintenance release that makes the central skills library scanner recursive, so skills nested inside subdirectories of the central directory are picked up automatically.

### Highlights

- Central skill scanning is now recursive: skills can live at any depth under the central directory (e.g. `~/skills/src/shared/<skill>/SKILL.md`), not only as immediate children.
- Scanner stops descending once it identifies a skill (a directory containing `SKILL.md`), so a skill's own subfolders are never re-scanned as separate skills.
- Hidden directories (anything starting with `.`) and well-known heavy directories (`node_modules`, `dist`, `build`, `target`, `__pycache__`, `.git`) are skipped to keep scans fast.
- Symlink loops are bounded by both a canonicalised visited-path set and a 16-level depth cap; dangling symlinks are silently ignored.

### Packages

- `skills-manage_0.9.2_windows_x64.msi`
- `skills-manage_0.9.2_windows_x64.zip`
- `skills-manage_0.9.2_macos_universal.dmg`
- `skills-manage_0.9.2_macos_universal.zip`
- `skills-manage_0.9.2_macos_universal.tar.gz`

### Notes

- This is a maintenance follow-up to `0.9.1`, scoped to scanner behaviour. No data migration is required.
- Skill IDs continue to be derived from the directory basename. If two skills with the same basename appear at different depths under the same central root, the later one wins on upsert (same behaviour as before, now simply reachable through the new recursive walk).
- Desktop builds remain unsigned, and macOS notarization is still deferred.

### Changelog

- `scan_directory` (`src-tauri/src/commands/scanner.rs`) walks recursively instead of one level deep.
- Added skip rules for hidden directories and `node_modules` / `dist` / `build` / `target` / `__pycache__` / `.git`.
- Added cycle protection through a canonicalised visited-path set and a hard depth cap (`MAX_SCAN_DEPTH = 16`).
- Replaced the legacy `test_scan_directory_is_not_recursive` assertion with three new tests covering recursive discovery, no-descend-into-a-skill, and skip rules.

## 中文

这是一次维护版本，把中央技能库的扫描器改为递归扫描，使中央目录下嵌套的技能也能被自动识别。

### 亮点

- 中央技能扫描改为递归：技能可以放在中央目录下的任意层级（例如 `~/skills/src/shared/<skill>/SKILL.md`），不再只识别"中央目录的直接子目录"。
- 一旦某目录被识别为技能（包含 `SKILL.md`），递归会在该目录止步，不会把技能内部的子目录再当成独立技能。
- 跳过隐藏目录（任何以 `.` 开头的目录）和常见重型目录（`node_modules`、`dist`、`build`、`target`、`__pycache__`、`.git`），保持扫描速度。
- 通过规范化路径的已访问集合 + 16 层深度上限来防 symlink 死循环；悬挂的 symlink 会被静默忽略。

### 安装包

- `skills-manage_0.9.2_windows_x64.zip`
- `skills-manage_0.9.2_macos_universal.dmg`
- `skills-manage_0.9.2_macos_universal.zip`
- `skills-manage_0.9.2_macos_universal.tar.gz`

### 说明

- 这是对 `0.9.1` 的维护补充版本，仅修改扫描行为，不需要任何数据迁移。
- 技能 ID 仍然由目录 basename 派生。如果同一中央根下不同层级出现重名 basename，upsert 时后写覆盖前写（行为与原先一致，只是现在能被递归扫到）。
- 当前桌面构建仍为未签名状态，macOS notarization 继续留待后续版本处理。

### 更新内容

- `scan_directory`（`src-tauri/src/commands/scanner.rs`）改为递归扫描，不再只走一层。
- 新增对隐藏目录和 `node_modules` / `dist` / `build` / `target` / `__pycache__` / `.git` 的跳过规则。
- 新增基于规范化路径的访问集合和深度上限（`MAX_SCAN_DEPTH = 16`）做循环保护。
- 移除原 `test_scan_directory_is_not_recursive` 断言，新增三项测试：递归发现、技能内部不再下钻、隐藏/重型目录跳过。
